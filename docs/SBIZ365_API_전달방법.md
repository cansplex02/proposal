# 소상공인365 API 3종 — 전달해 주실 내용

발급받으신 **상권지도 · 간단분석 · 상세분석** 을 하나씩 보내주시면 코드에 연결합니다.

## API마다 이것만 알려주세요

1. **API 이름** (상권지도 / 간단분석 / 상세분석)
2. **요청 URL** (전체 주소 또는 `https://bigdata.sbiz.or.kr` 뒤 경로)
3. **HTTP 방식** (보통 GET)
4. **필수 파라미터 이름**
   - 인증키: `serviceKey` 인지 다른 이름인지
   - 위도·경도: `lat`/`lng` 인지 `cx`/`cy` 인지
   - 반경: `radius` 단위가 **미터**인지
5. **샘플 응답** JSON (명세서에 있거나, Postman/브라우저로 1번 호출한 결과)

채팅에 그대로 붙여넣으셔도 되고, `data/sbiz365-api-spec.json` 에 저장하셔도 됩니다.  
템플릿: `data/sbiz365-api-spec.template.json`

## 분석 페이지에서 쓰일 위치 (예상)

| 365 API | 분석 HTML 영역 |
|---------|----------------|
| **상세분석** | 섹션 1 — 주거·직장 인구 표 |
| **간단분석** | 섹션 2 — 요약 카드, 미니 인사이트 |
| **상권지도** | 섹션 2 — 지도·상권 기본 정보 |

※ 실제 JSON 필드명은 명세를 받은 뒤 맞춥니다.

## 지오코딩(주소 → 좌표)은 365와 별도

**네이버 Maps Geocoding** (`NAVER_MAP_CLIENT_ID` / `NAVER_MAP_CLIENT_SECRET`) 또는 카카오 로컬 API로 좌표를 구한 뒤, 365 API(상권지도·간단·상세)에 `lat`/`lng`를 넘깁니다.
